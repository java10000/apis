aceona.user.search 用户信息检索

 

简要描述

用户信息检索

 

系统级输入参数

名称 类型 是否必需 描述
method String 是 API接口名称
timestamp String 是 时间戳，格式为yyyy-MM-dd HH:mm:ss，例如：2008-01-25 20:23:30。
format String 否 可选，指定响应格式。默认json,目前支持格式为xml,json
app_key String 是 分配给应用的AppKey
sign String 是 分配给应用的授权密钥

应用级输入参数

名称 类型 是否必须 描述 示例值 默认值
qString必须搜索条件深圳
page_noInteger必须页数1
page_sizeInteger必须单页返回数据数目15

返回结果

名称 类型 是否必须 示例值 描述

		AceonaClient client = new DefaultAceonaClient(Constants.AOP_SERVER_URL,"aceoner","SH!!!!!!!!!!!!!!!!");
		UsersSearchRequest req = new UsersSearchRequest();
		req.setPageNo(0L);
		req.setPageSize(15L);
		//accounts.login_name:'2aceona@aceona.com' 搜索login_name为2aceona@aceona.com
		//(accounts.login_name:'*@aceona.com' && user_type:'2') 搜索后缀为@aceona.com并且user_type为2的用户
		//(accounts.login_name:'*@aceona.com' && user_type:'2') || addresses.address:'深圳' 搜索后缀为@aceona.com并且user_type为2的用户，或者addresses.address中有深圳的用户
		//(accounts.login_name:'*@aceona.com' && user_type:'2') && accounts.update_time:['20100101':'20121212'] 搜索后缀为@aceona.com并且user_type为2的用户,并且在update_time指定的时间段类
		//(accounts.login_name:'*@aceona.com' && user_type:'2') - accounts.login_name:'2aceona@aceona.com' 搜索后缀为@aceona.com并且user_type为2的所有用户，但是要排除掉2aceona@aceona.com
		//(user_type:[0,2])获取0到2的数据
		//(user_type:{0,2])获取1、2的数据 {}为排除，[]为包含，可以组合使用
		
		req.setQ("accounts.login_name:'2aceona@aceona.com'");
		
		try {
			UserSearchResponse res = client.execute(req);
			System.out.println(res.getBody());			
			System.out.println(res.getElements());
			System.out.println(res.isSuccess());
			if(res.getElements()!=null){
				for(User user : res.getElements()){
					System.out.println(user.getId());
				}
			}
		} catch (ApiException e) {
			// TODO Auto-generated catch block

			System.out.println("ERROR!");
